This app does not ask the user to paste a CraftWorld JWT manually in the browser. Instead, the browser signs in with a Ronin wallet, the server exchanges that wallet proof for a CraftWorld/Firebase token, and then the browser sends that token back to the app’s API routes as a normal Authorization: Bearer ... token.

How API calls use the Firebase token

Browser side

Whenever the frontend needs to call an authenticated API route, it reads cw_token from local storage:
function getCwToken() {
  const raw = String(localStorage.getItem(CW_TOKEN_KEY) || '').trim();
  if (!raw) return '';
  if (raw.toLowerCase().startsWith('bearer ')) {
    return raw.slice(7).trim();
  }
  return raw;
}
That means the browser supports either a raw token or a string like Bearer <token>, but it normalizes it before use.
The generic authenticated fetch helper then attaches the token like this:
headers.set('Authorization', Bearer ${token});
the important rule is:
Authorization: Bearer <firebase idToken>
-



⸻

Server side

The backend extracts the token from the Authorization header using:
extract_bearer_token(request.headers.get("Authorization"))
That helper only accepts the standard Bearer <token> form.
Then the backend normalizes the token before sending it to CraftWorld. The normalizer does something important:
if value.startswith("jwt"):
    return value
if value.count(".") >= 2:
    return f"jwt{value}"
return value
So if the browser sends a Firebase JWT like:
eyJhbGciOi...
the server converts it to:
jwt_eyJhbGciOi...
before calling CraftWorld.
That is important because CraftWorld appears to expect the Firebase token to be prefixed with jwt_ when used as a CraftWorld bearer token.

⸻

How the server calls CraftWorld GraphQL

All low-level CraftWorld GraphQL calls eventually build headers like this:
headers = {
    "Authorization": f"Bearer {normalized_token}",
    "Content-Type": "application/json",
    "x-app-version": CW_APP_VERSION,
}
Then the server posts JSON to:
https://craft-world.gg/graphql
with
{
  "query": query,
  "variables": variables
}
That is implemented in call_graphql_with_jwt.
The higher-level call_graphql helper either uses the per-user bearer token passed by the browser or falls back to the server environment variable CRAFTWORLD_JWT. For browser-authenticated calls, you want the per-user bearer_token path.
If CraftWorld returns GraphQL errors, the app raises an error instead of silently continuing.

How the app gets the CraftWorld UID

To call fetchCraftWorld(uid: ID!), you need a CraftWorld UID.

During Firebase sign-in, after the backend receives the idToken, it tries to resolve the CraftWorld UID by calling CraftWorld account identity data. If that fails, it falls back to decoding the JWT payload.
The reusable identity service does the same thing:
    
Normalize token.
Call CraftWorld account identity queries.
Try extracting UID from account.uid, account.id, linked accounts, or trade account.
Fall back to extracting UID from the JWT payload.
.

The UID extraction helper can pull the UID from direct fields, linked account details, trade account fields, or JWT payload fields such as uid, user_id, or sub.
.
The frontend also has an /api/account_uid call. It uses the generic authFetch, so the token is automatically sent as Authorization: Bearer <token>.
.

The /api/account_uid route is registered along with /api/account_proficiencies and /api/account_workshop.

Option B: Call CraftWorld GraphQL directly with the user’s Firebase token

If your already has the Firebase idToken, you can call CraftWorld directly.

The key rules are:
    
Use endpoint:
 
https://craft-world.gg/graphql


Add headers:
Content-Type: application/json
x-app-version: 1.11.0
Authorization: Bearer jwt_<firebase idToken>
If the token already starts with jwt, do not double-prefix it.
If the token is a normal JWT with two dots, prefix it with jwt

That is exactly what this app’s token normalizer does.

Example JavaScript: 
function normalizeCraftWorldToken(token) {
  const value = String(token  '').trim();
  if (!value) throw new Error('Missing token');

  if (value.startsWith('jwt_')) return value;

  // Firebase JWTs normally have 3 dot-separated parts.
  if ((value.match(/./g)  []).length >= 2) {
    return jwt_${value};
  }

  return value;
}

async function fetchCraftWorldConfig({ idToken, uid }) {
  const token = normalizeCraftWorldToken(idToken);

  const query = 
    query FetchCraftWorld($uid: ID!) {
      fetchCraftWorld(uid: $uid) {
        landPlots {
          areas {
            symbol
            factories {
              factory {
                level
                definition { id }
              }
            }
          }
        }
        mines {
          level
          definition { id }
        }
        dynos {
          meta { displayName rarity }
          production { amount symbol }
        }
        resources {
          symbol
          amount
        }
      }
    }
  ;

  const res = await fetch('https://craft-world.gg/graphql', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-app-version': '1.11.0',
      Authorization: Bearer ${token},
    },
    body: JSON.stringify({
      query,
      variables: { uid },
    }),
  });

  const body = await res.json();

  if (!res.ok  body.errors) {
    throw new Error(JSON.stringify(body.errors  body, null, 2));
  }

  return body.data.fetchCraftWorld;
